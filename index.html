Задачи

  1. Ускорить, упростить процесс публикации документации
  2. Снизить болезненность внесения изменений

Проблемы

  1. ASP.NET MVC как фундамент документации
  2. SSR, in-memory rendering
  3. HTML-like формат как способ хранения контента
  4. Отсутствие стандартизации, консистентности UI
  5. Отсутствие стандартов в "общении" между Командой Интеграции и командами продуктов
  6. Непропорциональное количество ответственности на Команде Интеграций

Решения

  1. Отказ от C#, переход на NodeJS
  2. Статические страницы на смену SSR
  3. Markdown как способ хранения контента
  4. Начало формирование единой дизайн-системы её компонентов
  5. Выработка стандартов в "общении" между командами
  6. Более точное распределение ответственности

Итог
